5 questions for Monday

Before you add AI to a feature, decide where the intelligence lives. Run the feature through these — the answer is an L-number, not "add a chatbot."
L2 plain chat L3 components as tools L4 catalog + composition L5 route to apps L6 shared state
1
Is our agent's output one type of UI, or many?
One known shape → L3 is enough. Many unpredictable shapes → consider L4.
2
Does the agent need to modify content the user is actively editing?
Yes → L6 (shared state). Don't fake it by bolting L3 onto this need.
3
Do we need to build the UI, or can we route to one that exists?
Embeddable already? → L5. Embed it; don't reinvent the sketchpad.
4
Can the user predict what UI they'll see each time?
Yes → lean L3 (visual consistency wins). No → accept L4's flexibility cost.
5
When the agent fails, can the user fall back to a familiar UI?
There must be a fallback. L3 + a text fallback is the default safety net.
Chat is a feature.
UI is the product.
Don't bolt on. Redesign.
The ask

Monday morning. One sentence. Out loud.

Make the structural bet visible — and make someone disagree with you.
Walk into your standup, 1:1, or team Slack and say:

"Feature   should be L  because  ."

Not "we should use AI more." Not "we should be more agentic." Those die in a roadmap doc. Say the L-number — that's how this stops being a slide deck and starts being a product decision.
